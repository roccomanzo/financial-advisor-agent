# Agent Spec - Financial Advisor Agent

## Overview
A financial advisor agent designed to guide users through personal financial decisions. It provides structured guidance on retirement planning, saving for major purchases (e.g., house or car), managing wealth/windfalls, living in retirement or on a fixed income, and debt management.

The agent works in prototype mode (local playground) and utilizes financial data APIs (Google Finance or Yahoo Finance) to fetch stock/market information, while adhering to strict privacy and safety guardrails.

## Example Use Cases
### Use Case 1: Retirement Planning
*   **Input**: User specifies their current age (e.g., 35), current savings, monthly contributions, and target retirement age (e.g., 65).
*   **Output**: A projection of their retirement nest egg, analysis of the shortfall or surplus, and a step-by-step strategy to bridge any gaps.

### Use Case 2: Saving for a Major Purchase
*   **Input**: User wants to buy a house in 5 years, requiring a $100,000 down payment.
*   **Output**: A savings schedule, suggested asset allocation for short-to-medium term savings, and tips to optimize savings.

### Use Case 3: Managing Debt
*   **Input**: User lists outstanding debts (credit card: $5,000 at 20% APR, auto loan: $15,000 at 5% APR).
*   **Output**: A debt payoff plan comparing Snowball and Avalanche methods, and monthly allocation recommendations.

## Tools Required
1.  **Google Finance / Yahoo Finance API Tool**:
    *   **Purpose**: Fetch current stock/ETF prices, historic performance, and market indexes to assist with investment/savings allocations.
    *   **Auth**: No auth required for public scraping/unauthenticated endpoints, or standard API key.
2.  **Financial Calculator Tool**:
    *   **Purpose**: Python-based math tool for compounding interest, loan amortization schedules, and present/future value calculations.

## Constraints & Safety Rules
1.  **NO PII Access**: The agent must not request, store, or process any personally identifiable information (e.g., SSN, full bank account numbers, real names, physical addresses).
2.  **NO Real Transactions**: The agent cannot execute real trades or move funds. All recommendations are simulation-only.
3.  **Required Disclaimer**: Every financial projection or recommendation must be accompanied by a clear disclaimer stating that this is not professional financial, tax, or legal advice.
4.  **No Direct Banking Credentials**: The agent will rely on user-entered values (mock or real amounts/rates) rather than authenticating directly to real banking institutions.

## Success Criteria
*   Correctly calculates compound interest and amortization schedules.
*   Provides structured, actionable, and non-prescriptive plans for retirement, major purchases, windfalls, and debt.
*   Gracefully falls back from Google Finance to Yahoo Finance API if the primary source fails.
*   Consistently appends safety disclaimers to responses.

## Architecture & Sub-Agents
To handle distinct financial scenarios effectively, the root advisor agent will delegate to specialized sub-agents:
1.  **Retirement & Windfall Agent**: Focuses on long-term compound growth, retirement drawdown strategies, and structured windfalls.
2.  **Savings & Purchase Agent**: Focuses on short-to-medium term goals, down payments, and budgeting.
3.  **Debt & Liability Agent**: Focuses on optimization of interest payments, credit card payoff strategies, and loan amortization.
4.  **Market Data Fetcher (Tool-Helper)**: Specialized tool calling to retrieve asset prices and market metrics.
